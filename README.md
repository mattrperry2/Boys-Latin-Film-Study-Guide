# Defensive Facility — v2

Sinatra + Postgres. Coach assigns plays + Hudl link + formation + gap.
Player taps a play, picks their two pre-snap reads, hits "Lock In".
That's it.

## Files
- `app.rb` — the whole app
- `Gemfile` / `Gemfile.lock` — dependencies (Gemfile.lock generated by you, see below)
- `config.ru` — Rack entry point for Puma
- `Procfile` — start command for Render
- `.ruby-version` — pins Ruby
- `.gitignore`

## First-time setup before pushing to GitHub

Render REQUIRES `Gemfile.lock` to be checked in. Generate it on your laptop:

```bash
cd /path/to/this/folder
bundle install
git add Gemfile.lock
```

If you don't have bundler:
```bash
gem install bundler
```

Then commit and push all files.

## Render environment variables

- `DATABASE_URL` — connection string for your Postgres database
  (use the **Internal Database URL** from your Render Postgres dashboard)

## Build & start commands on Render

- **Build command:** `bundle install`
- **Start command:** `bundle exec puma -e production -p $PORT`
  (or leave it blank and Render will use the Procfile)
